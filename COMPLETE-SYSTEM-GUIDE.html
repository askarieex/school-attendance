Super Admin Panel

Purpose: Platform management for YOUR team
  • Manage all schools - Create, edit, deactivate
  • Device management - Generate API keys
  • User management - Create school admin accounts
  • System analytics - Platform-wide metrics
  • Billing & subscriptions - Revenue tracking
Access: Only your internal team can access this

School Admin Dashboard

Purpose: Daily operations for CLIENT schools
  • Live dashboard - Real-time attendance today
  • Student management - Add, edit, import students
  • Reports - Daily, weekly, monthly exports
  • Notifications - SMS alerts to parents
  • School settings - Timings, late threshold
Access: Each school ONLY sees their own data

Why Separate Dashboards?

✓ Security: Super admin can't accidentally access client interface
✓ User Experience: Each sees only what they need
✓ Branding: School dashboard can be white-labeled
✓ Development: Easier to maintain two focused apps
✓ Scalability: Deploy separately, scale independently
✓ Customization: Schools can have custom themes

Multi-Tenancy: The school_id Pattern

Notice school_id appears in: users, devices, students, attendance_logs, school_settings

-- Example: School Admin queries their students SELECT * FROM students WHERE school_id = 5 -- Automatically filtered by middleware AND is_active = TRUE; -- Super Admin sees ALL schools SELECT school_id, COUNT(*) FROM students GROUP BY school_id; -- No school_id filter for super admin

Complete System Flow

Step-by-Step: How Everything Works Together

Flow 1: Student Scans RFID Card

Step 1: Physical Scan

Student arrives at school and scans their RFID card at entry point

RFID Card ID: "ABC123456" Time: "2025-01-15T08:12:30Z"

Step 2: Device Sends to API

RFID hardware device sends HTTP POST request to backend

POST /api/v1/attendance/log Headers: X-API-Key: "device-abc-123-xyz" Body: { "rfid_card_id": "ABC123456", "timestamp": "2025-01-15T08:12:30Z" }

Step 3: Backend Validates Device

API checks if device API key is valid and active

-- Database query SELECT id, school_id FROM devices WHERE api_key = 'device-abc-123-xyz' AND is_active = TRUE; -- Result: device_id = 7, school_id = 5

Step 4: Find Student

Lookup student by RFID card ID and verify they belong to same school

SELECT id, full_name, parent_phone, school_id FROM students WHERE rfid_card_id = 'ABC123456' AND school_id = 5 -- Security: must match device's school AND is_active = TRUE; -- Result: student_id = 42, name = "Emma Johnson"

Step 5: Calculate Status

Compare check-in time against school start time to determine if late

-- Get school settings School Start Time: 08:00 AM Late Threshold: 15 minutes Check-in Time: 08:12 AM Status: "present" (within 15 min threshold) -- If check-in was 08:20 AM: Status: "late" (20 min > 15 min threshold)

Step 6: Save Attendance Log

Insert record into attendance_logs table

INSERT INTO attendance_logs ( student_id, school_id, device_id, check_in_time, status, date ) VALUES ( 42, 5, 7, '2025-01-15 08:12:30', 'present', '2025-01-15' );

Step 7: Send SMS to Parent

If SMS enabled, trigger notification to parent's phone

// Twilio API call sendSMS({ to: "+1234567890", message: "Emma Johnson arrived at school at 8:12 AM - Greenwood High" }); // Update attendance log UPDATE attendance_logs SET sms_sent = TRUE WHERE id = 12345;

Step 8: Response to Device

API returns success response to hardware device

HTTP 200 OK { "success": true, "student_name": "Emma Johnson", "status": "present", "time": "08:12 AM" } // Device displays: "Welcome Emma ✓"

Flow 2: School Admin Views Dashboard

Step 1: Admin Logs In

POST /api/v1/auth/login { "email": "admin@greenwood.edu", "password": "********" } // Response: { "token": "eyJhbGci...", "user": { "id": 10, "role": "school_admin", "school_id": 5, ← Critical for multi-tenancy "school_name": "Greenwood High" } }

Step 2: Frontend Requests Dashboard Stats

GET /api/v1/school/dashboard/today Headers: Authorization: Bearer eyJhbGci... ← JWT contains school_id

Step 3: Backend Validates & Extracts school_id

// Auth middleware decodes JWT const decoded = jwt.verify(token); // decoded = { userId: 10, role: 'school_admin', schoolId: 5 } // Attach to request req.user = decoded; req.schoolId = 5; ← This is how multi-tenancy works!

Step 4: Query Database (Filtered by school_id)

-- Query 1: Total students SELECT COUNT(*) FROM students WHERE school_id = 5 AND is_active = TRUE; -- Result: 542 -- Query 2: Today's attendance breakdown SELECT status, COUNT(*) as count FROM attendance_logs WHERE school_id = 5 ← Automatic filtering AND date = '2025-01-15' GROUP BY status; -- Results: -- present: 495 -- late: 12 -- Total checked in: 507 -- Absent: 542 - 507 = 35

Step 5: API Returns Data

{ "total_students": 542, "present": 507, "absent": 23, "late": 12, "attendance_rate": 93.5, "last_updated": "2025-01-15T09:45:00Z" }

Step 6: Frontend Updates UI

React displays stat cards and sets up 30-second polling

// React useEffect hook useEffect(() => { fetchDashboardStats(); // Poll every 30 seconds const interval = setInterval(() => { fetchDashboardStats(); }, 30000); return () => clearInterval(interval); }, []);

Complete API Structure & Innovation

All Endpoints + New Feature Ideas

Complete API Endpoints

Authentication APIs

POST /api/v1/auth/login
Login for both Super Admin and School Admin (returns JWT with role & school_id)
POST /api/v1/auth/refresh
Refresh expired access token
POST /api/v1/auth/forgot-password
Send password reset email

Super Admin APIs (Protected: role='superadmin')

School Management
GET/api/v1/super/schools
List all schools (pagination, search, filter by status)
POST/api/v1/super/schools
Create new school account
GET/api/v1/super/schools/:id
Get specific school details + stats
PUT/api/v1/super/schools/:id
Update school information
DELETE/api/v1/super/schools/:id
Deactivate/suspend school
Device Management
POST/api/v1/super/devices
Generate new device API key for school
GET/api/v1/super/devices
List all devices across all schools
DELETE/api/v1/super/devices/:id
Revoke device API key
Platform Analytics
GET/api/v1/super/stats/platform
Platform-wide metrics (total schools, students, revenue)
GET/api/v1/super/audit-logs
System audit trail (all admin actions)

School Admin APIs (Protected: role='school_admin' + school_id filter)

Student Management
GET/api/v1/school/students
List students in MY school (auto-filtered by school_id from JWT)
POST/api/v1/school/students
Add new student to MY school
POST/api/v1/school/students/import
Bulk import students via CSV
GET/api/v1/school/students/:id
Get student details + attendance history
PUT/api/v1/school/students/:id
Update student information
Dashboard & Reports
GET/api/v1/school/dashboard/today
Real-time stats for today (present, absent, late counts)
GET/api/v1/school/dashboard/recent-checkins
Last 20 check-ins (for live feed)
GET/api/v1/school/reports/daily?date=YYYY-MM-DD
Attendance report for specific date
GET/api/v1/school/reports/export?format=csv|pdf
Export attendance data
Settings
GET/api/v1/school/settings
Get school settings (start time, late threshold, SMS config)
PUT/api/v1/school/settings
Update school settings

Hardware Device API (Protected: X-API-Key header)

POST/api/v1/attendance/log
RFID device posts attendance scan (validates device, logs attendance, triggers SMS)
GET/api/v1/attendance/verify/:rfid
Verify if RFID card exists (returns student name for device display)

NEW Innovative Features

QR Code Check-In

Alternative to RFID: Students scan personal QR code on phone

  • Lower hardware cost (just tablets/phones)
  • Parents get unique QR for their child
  • Dynamic QR changes daily (more secure)
  • Works offline (syncs when online)

AI Attendance Predictions

Machine Learning: Predict chronic absenteeism

  • Alert admin if pattern detected
  • Identify at-risk students early
  • Suggest intervention strategies
  • Trend analysis across grades

Bus Tracking Integration

Complete journey: From home to school

  • Scan when boarding bus
  • Track bus location (GPS)
  • ETA notifications to parents
  • Attendance marked on bus scan

Health Screening Module

Post-pandemic feature: Temperature checks

  • Thermal scanner at entry
  • Auto-log temperature with attendance
  • Alert if fever detected
  • Health compliance reports

Emergency Lockdown Mode

Safety feature: Know who's on campus

  • One-click lockdown activation
  • Instant list of present students
  • Send emergency alerts to parents
  • Export current location data

Parent Mobile App

Dedicated app: Better than SMS

  • Push notifications (cheaper than SMS)
  • View child's attendance history
  • Request absence approval
  • Chat with school admin
  • iOS + Android apps

Smart Absence Alerts

Intelligent notifications: Context-aware

  • Different alert if student is chronically late
  • Escalate to principal if 3+ days absent
  • WhatsApp integration (global reach)
  • Customizable alert rules per school

Class-by-Class Attendance

Beyond entry: Track each period

  • Teachers mark attendance per class
  • Tablet in each classroom
  • Detect students skipping specific classes
  • Comprehensive daily timeline

Gamification for Students

Motivate attendance: Make it fun

  • Badges for perfect attendance
  • Monthly leaderboards (class-level)
  • Rewards program integration
  • Privacy-friendly (no individual shaming)
